### 자료형

- 정의: 변수에 저장되는 데이터의 종류
- 컴퓨터는 멍청해서 어떤 자료형을 사용해야 할지 모르기 때문에 자료형에 대한 명시 필요
- 종류
    - int(정수): 소수점이 없는 음수, 0, 양수
    - float(실수): 소수점이 있는 숫자(부동소수점 오차 없애려면 decimal 사용)
    - str(문자열): 따옴표로 감싸진 문자들의 나열
    - bool(논리형): True(1) 또는 False(0)
    - None: 비어있는 게 아닌 값 자체가 없음
 - 캐스팅 = 형변환 = 데이터의 자료형을 다른 자료형으로 바꿈

### 컬렉션 자료형

- 여러 개의 값을 담는 자료형

리스트
- 여러 개의 값을 순서대로 저장
- 자료형 중 가장 유연
- CRUD
  - `리스트.append("요소")` - 맨 뒤에 추가
  - `리스트.insert(1,"요소")` - 특정 위치에 끼워넣기
  - `print(리스트[0])` - 읽기
  - `리스트[0] = "요소"` - 바꾸기
  - `리스트.remove("요소")` - 해당 값 삭제(데이터 삭제는 리스크가 있기 때문에 따로 빼놓는 것이 좋음)
  - `리스트.pop()` - 마지막 값 꺼내고 삭제

---
 
딕셔너리
- 키와 값을 붙여 저장
- 이름으로 탐색
- CRUD
  - `딕셔너리["키"] = "값"` - 추가하기
  - `딕셔너리.update({"키":"값", "키":"값"})` - 여러 개 한번에 추가(다른 요소들과 동급으로 들어감)
  - `print(딕셔너리["키"])` - 읽기
  - `딕셔너리["키"] = "값"` - 수정하기
  - `del 딕셔너리["키"]` - 삭제하기(키 이름을 알 때)
  - `딕셔너리.pop("키")` - 삭제하면서 값 반환
  - `딕셔너리.clear` - 전부 삭제
- 없는 키는 get() 사용하면 안전(없음 말고~)

문자열
- 인덱싱 가능
- 자주 쓰는 기능
  - `print(text.upper())` - 대문자 변환
  - `print(text.lower())` - 소문자 변환
  - `print(text.strip())` - 앞뒤 공백 제거
  - `print(text.replace("바꿀_단어", "바뀔_단어"))` - 글자 바꾸기
  - `text.split("분기_기준")` - 문자열 나누기
  - `print(f"문자1 {변수1} 문자2 {변수2}")` - f-string

튜플
- 한 번 만들면 수정할 수 없는 리스트
- 소괄호()로 표현
- 데이터가 바뀌면 안될 떄 사용
- 수정, 추가, 삭제는 안되지만 읽기는 가능
- 인덱싱 존재 => 언패킹 가능 = 튜플의 값을 여러 변수엫 한 번에 담기 가능

집합
- 중복을 허용하지 않고 순서가 없는 자료형
- 순서가 없어 인덱싱과 수정은 안되지만 추가, 읽기, 삭제 가능
  - `집합.add(요소)` - 추가하기
  - `집합.update([요소1, 요소2, ...])` - 한번에 여러 개 추가
  - `집합.remove(요소)` - 값 삭제
  - `집합.discard(요소)` - 값 삭제(없음 말고~)

### 함수
- 동작을 미리 만들어 놓은 것(과정은 몰라도 됨 like pirnt)
- 코드 재사용 / 가독성 향상 / 유지보수 용이 / 분업 가능
- 변수
  - 매개변수: 함수 정의 시 사용하는 변수 이름
  - 인자: 함수 호출 시 실제로 전달하는 값
  - 범위
    - 지역 변수: 함수 내부에서 만든 변수, 함수 끝나면 사라짐
    - 전역 변수: 함수 바깥에서 만든 변수. 프로그램 전체에서 사용 가능
    - 함수 내부에서 같은 이름의 변수를 만들면 새로운 지역변수 생성(덮어씀 X) 
- return이 종료조건이며 return 조건이 여러개 있을 수 있음.
- 매개변수에 기본값을 설정할 수 있음(인자 생략)
- 인자를 이름으로 지정해 순서 상관없이 전달 가능
- caller: 부르는 놈, callee: 일을 수행하는 쪽
- LEGB: 안쪽에서 시작해 프로그램이 실행되면서 날라감(함수 내부->외부 함수->모듈 전역->파이썬 내장)
- global: 함수 내부에서 전역변수 수정할 때 사용하지만 사용 시 변수 움직임이 예측되지 않으므로 웬만하면 쓰지 않는 것이 좋음

### class
- 정의: 관련 데이터와 기능을 하나로 묶어 코드로 정리해 여러개를 찍어낼 수 있는 설계도
- 상태를 관리하는데 적합
- 좋은 클래스는 한 클래스 당 한 가지 역할만
- `__init__` 메서드
  - 인스턴트 생성 시 자동 실행
- `self`
  - 정의: 나 자신
  - 자신의 속성에 접근하기 위해 사용
  - self 없으면 그냥 지역변수일 뿐
  - __init__ 함수의 첫번째 인자, 다른 이름을 붙여도 되나 약속함
  - dataclass
    - 단순 데이터 저장용 클래스
    - __init__ 자동 생성
    - print() 했을 때 가시성 굿
    - 코드 간결
    - 타입 힌트: 변수의 타입을 알려줌
```python
from dataclasses import dataclass

@dataclass
class ServerConfig:
    host: str
    port: int
    timeout: int = 30  # 기본값 설정 가능

config = ServerConfig("localhost", 8080)
print(config)
```
